Опис проєкту

Цей проєкт реалізує генератор документа з реактивним шаром на основі патерну Observer (Спостерігач). Під час рендерингу елементів документа (Section, Paragraph, List) система генерує події, які передаються підписникам через центральний об’єкт RenderEventPublisher.

Мета роботи
Метою є розширення існуючого генератора документа можливістю відстеження процесу рендерингу та збору додаткової інформації без зміни логіки самих елементів.

Архітектура рішення
RenderEventPublisher — центральний статичний клас, який керує підписниками та розсилає події
RenderEventSubscriber — інтерфейс підписників
RenderContext — структура даних події рендерингу

Підписники:
RenderLoggerSubscriber — логування рендерингу елементів
SummaryCollector — збір статистики по елементах документа
PerformanceSubscriber — вимірювання загального часу рендерингу

Принцип роботи
Після завершення рендерингу кожного елемента документа створюється об’єкт RenderContext, який передається в метод notify() класу RenderEventPublisher. Далі всі підписники отримують оновлення через метод update().

Приклад використання
У файлі main.ts підписники підключаються наступним чином:
створюються екземпляри підписників
вони підписуються через RenderEventPublisher.subscribe()
запускається процес генерації документа

Очікуваний результат
Під час виконання програми в консолі відображаються:
логи кожного відрендереного елемента
підсумкова статистика кількості елементів
загальний час рендерингу
